# Routes & Content Specs

This folder defines URL structures and the content that powers them so AIX can route correctly and preserve SEO/localization rules.

- **Role:** Specify route maps, params, redirects, content sources, rendering/caching, localization, and SEO behaviors.
- **What lives here:** Route definitions, content-source mappings (CMS/API/static), rendering/caching rules, locale strategy, SEO/meta rules, decisions, and open questions.
- **Consumers:** AIX (for routing/content guidance), frontend engineers, SEO/UX stakeholders.
- **Source links:** Reference CMS/data in `/specs/cms/` and `/specs/data/`, performance budgets in `/specs/performance/`, and ADRs in `/docs/decisions/`.
